# Mission: Understand RLHF deeply enough to teach it

## Why
Build a clear, durable mental model of reinforcement learning and RLHF — strong
enough to explain it in my own words. The concrete output is a beginner-friendly
blog series ("RL for Beginners" / "RLHF Explained") that teaches what I've learned.
Teaching it back is both the goal and the method. Strong technical foundations as for RLHF

## Success looks like
- I can read any slide, figure, or section of the RLHF Book and explain it plainly.
- I can write a clear blog post explaining each core RLHF concept to a beginner.
- I hold an accurate mental model of the full pipeline: base model → SFT → reward
  model → RL optimization, plus the key algorithms (PPO/GRPO, DPO).
- I can open an RLHF research paper and follow its core argument without drowning.

## Constraints
- New to AI/ML; strong-but-rusty math (engineer, BITS Pilani, 99%ile JEE).
  Knows probability basics; has forgotten calculus/integration — refresh on demand.
- Can read/write basic Python; no ML libraries (numpy/pytorch) yet.
- Time is **not** the constraint. The priority is the fastest path to *real* understanding.
- Intuition first, then gentle rigor. Enough math to genuinely understand — never hand-waving.

## Out of scope (for now)
- Becoming a publishing researcher / deriving novel methods from scratch.
- Production-scale training infrastructure and distributed systems engineering.

## Method note
Each lesson is gated by a clear, three-rung **Mastery Check** (Recall → Apply → Explain);
you pass only by clearing Apply + Explain, so passing means you genuinely own the concept
and failing pinpoints exactly what to re-teach. The **blog post** is the mission *output*
(written once a concept is passed, collected at the end of each phase) — not the gating
test. Math is taught at the moment each concept needs it, with every symbol explained,
never front-loaded.
