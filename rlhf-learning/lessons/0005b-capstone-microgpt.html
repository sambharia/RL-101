Warm-up: pull it all back

This is the last recall before the rebuild. Every question draws on a different lesson.

SR1 (Lesson 2) — What is next-token prediction? How does a language model generate text?

A language model takes in a sequence of tokens, runs them through the network, and outputs a probability distribution over the next token via softmax. To generate text, you sample one token from that distribution, append it to the input, and repeat until a stop condition.

SR2 (Lesson 3) — What is the training loop? What does the loss $-\log P(y_\text{correct})$ measure?

Forward pass → compute loss → backward pass (backpropagation) → update parameters. The loss $-\log P(y_\text{correct})$ measures how surprised the model is by the correct answer — high loss means the model assigned low probability to the right token. The gradient pushes the model to assign more probability to the correct token.

SR3 (Lesson 3b) — Name the components of one transformer block in order. What does each do?

(1) LayerNorm/RMSNorm — normalize the input. (2) Multi-head attention — Q/K/V projections, dot-product scores, softmax weights, weighted sum of Values, output projection. (3) Residual add — add the input back. (4) LayerNorm/RMSNorm again. (5) MLP — expand (fc1), nonlinearity (ReLU/GELU), compress (fc2). (6) Residual add — add the input back again.

SR4 (Lesson 5) — Name the six core RL terms. Then state the RL objective in one sentence.

Agent (decision-maker), state (what it observes), action (what it does), reward (feedback), policy (strategy — $\pi(a|s)$), return (total reward). RL objective: find the policy that maximizes expected return.

Hint: Block 2 — Autograd (Value class)

Each Value stores: its numerical data, its gradient (starts at 0), its children in the computation graph, and the local derivatives (how this node's value changes w.r.t. each child). The backward() method: topologically sort the graph, set the root's gradient to 1, then walk in reverse order — for each node, multiply the node's gradient by each local derivative and add it to the child's gradient. Key derivatives: $\frac{d}{dx}(x+y) = (1, 1)$, $\frac{d}{dx}(x \cdot y) = (y, x)$, $\frac{d}{dx}(x^n) = n \cdot x^{n-1}$, $\frac{d}{dx}\log(x) = 1/x$, $\frac{d}{dx}e^x = e^x$, $\frac{d}{dx}\text{relu}(x) = \mathbb{1}[x > 0]$.

Hint: Block 5 — The GPT forward pass

The function takes one token at a time (not a batch). It processes: (1) Look up token embedding + position embedding, add them. (2) For each layer: RMSNorm → compute Q, K, V via linear projections → for each attention head, slice Q/K/V into head-sized chunks, compute attention scores (Q·K / sqrt(d)), softmax, weighted sum of V → concatenate heads → output projection → residual add → RMSNorm → MLP (fc1 → relu → fc2) → residual add. (3) Final linear projection (lm_head) to get logits over the vocabulary. The function also maintains a KV cache (keys and values from previous positions) so attention can look at all previous tokens.

Hint: Block 6 — Training loop

For each training step: (1) Grab a document, tokenize it (surround with BOS). (2) For each position in the sequence: feed the current token + position into gpt(), get logits, softmax to probabilities, compute loss as $-\log P(\text{next token})$. (3) Average all position losses. (4) Call backward() on the average loss. (5) Update every parameter using Adam: first moment $m$, second moment $v$, bias correction, then $\theta \leftarrow \theta - \text{lr} \cdot \hat{m} / (\sqrt{\hat{v}} + \epsilon)$. (6) Zero all gradients for the next step.
