Inside the Transformer

We've been running gpt() as a black box. Time to open it — and discover why it works.

Before we open the black box, recall what's around it. Don't peek — write your answer first, then reveal.

Q1 (Lesson 3) — What does the training loop do to the model's parameters at each step? Name the five-stage loop.

Forward → compute the model's prediction → Loss → measure how wrong it was (cross-entropy: $-\log P(y_\text{correct})$) → Zero grad → clear stale gradients → Backward → backprop: compute $\frac{\partial L}{\partial \theta_i}$ for every parameter → Update → nudge every parameter downhill: $\theta \leftarrow \theta - \eta \frac{\partial L}{\partial \theta}$.

Q2 (Lesson 2 + 3) — What does gpt() take as input, and what does it return? What type is each?

Input: a list of token IDs — integers (e.g. [15496, 11, 314]). Output: logits — one floating-point number per vocabulary item, at each position. Softmax turns logits into a probability distribution. The model doesn't know what "Paris" means; it only sees the integer ID for it.

1. The problem with integers

Every lesson so far, we've fed gpt() a list of token IDs — integers like 15496 for "Hello" or 2318 for "cat." The model returns logits. We've never asked: what happens inside?

Let's start with a simple, uncomfortable fact.

The integer problem

Token IDs are arbitrary integers. The ID for "cat" is 2318. The ID for "dog" is 3290. The ID for "kitten" is 19090. Those numbers don't mean anything — "cat" and "kitten" are semantically close, but |19090 − 2318| = 16772 is a large number. Arithmetic on IDs is meaningless.

Yet the transformer has to do math on these. It has to learn that "cat" and "kitten" are similar. How?

This is the first problem the transformer has to solve before it does anything else. And the solution is one of the simplest ideas in all of machine learning.

2. Embeddings: giving each token a meaningful address

Instead of using the integer ID directly, the model keeps a big lookup table called the embedding matrix (named wte in microgpt.py — "weight token embedding"). Its shape is $V \times d$: one row per vocabulary item, each row is a vector of $d = 768$ floating-point numbers.

Token ID → row index → 768-dimensional vector. That's it. The embedding of "cat" is just the 2318th row.

2316"cat"[ 0.28, −0.09, 0.74, 0.08, … ]
2318"cat"[ 0.29, −0.11, 0.73, 0.07, … ]

Notice the pattern in the example: "cat," "dog," and "kitten" have similar vectors. Not because we designed them that way — because the model learned these vectors from training data, and gradient descent naturally pushes semantically similar tokens towards similar embedding vectors. The rows of wte are parameters — they get updated by backpropagation exactly like any other weight.

The surprising consequence

Once training is done, you can do meaningful arithmetic on embedding vectors. The famous example: vec("king") − vec("man") + vec("woman") ≈ vec("queen"). The 768-dimensional geometry encodes semantic relationships. The model never explicitly learned this — it emerged from predicting the next token on trillions of words.

So: integer ID → embedding lookup → 768-dimensional vector. Now the model has something it can do math on. But there's a second problem.

3. Position matters: the second embedding

Consider these two sentences:

"The dog bit the man" "The man bit the dog"

Same tokens. Completely different meaning. The only difference is position. Yet if we just looked up each token's embedding, we'd get the same four vectors, in a different order — and the transformer's attention mechanism (coming next) is actually permutation-equivariant: if you shuffle the inputs, the outputs shuffle identically. Position would be completely invisible.

The fix: add a second lookup table, the positional embedding matrix (wpe — "weight position embedding"). Its shape is $T_{\max} \times d$ where $T_{\max} = 1024$ is the maximum sequence length. Row 0 is "the vector for position 0," row 1 is for position 1, and so on.

The final representation for each token is simply:

Input to the transformer stack
$$x_i = \underbrace{\text{wte}[\text{token\_id}_i]}_{\text{token embedding}} + \underbrace{\text{wpe}[i]}_{\text{position embedding}}$$
$\text{wte}[\cdot]$ Row lookup in token embedding matrix — shape $(768,)$.
$\text{wpe}[i]$ Row lookup in positional embedding matrix — shape $(768,)$, one per position $i$.
$x_i$ A single 768-dimensional vector for the $i$-th token, encoding both what it is and where it sits.

For a sequence of $T$ tokens, we get a matrix $X$ of shape $(T, 768)$. This is what enters the transformer blocks.

Both wte and wpe are learned. The model figures out how to encode position in 768 numbers all on its own, through training.

Now we have a rich vector for each token. The transformer's job is to update these vectors by letting each token look at the others. Which brings us to the hard part.

4. The contextualization problem: self-attention

Here's the issue. The word "bank" has one embedding — one point in 768-dimensional space. But "bank" means completely different things in:

"She walked to the river bank to fish." "She called her investment bank about the loan."

A fixed embedding for "bank" can't capture both senses. The model needs to update each token's representation by looking at the surrounding tokens. "River" and "fish" should push "bank" toward its geographical meaning. "Investment" and "loan" should push it toward its financial meaning.

More generally: what the word at position $i$ means depends on which other words are nearby. The transformer needs a mechanism for every token to gather information from every other token it's relevant to.

The question becomes: for each pair of tokens, how much should one attend to the other?

Queries, Keys, and Values

The attention mechanism works by giving every token three different roles simultaneously:

  • Query (Q): "What am I looking for?" — the question this token is asking.
  • Key (K): "What do I advertise?" — the label this token holds up to other queries.
  • Value (V): "If you pick me, here's what I contribute" — the actual content this token passes along.

Each token's embedding gets linearly projected into all three roles:

Projecting into Q, K, V
$$Q = X W_Q, \quad K = X W_K, \quad V = X W_V$$
$X$ Input matrix, shape $(T, 768)$ — one row per token.
$W_Q, W_K, W_V$ Learned weight matrices, each shape $(768, 64)$. These are parameters — updated by gradient descent.
$Q, K, V$ Each shape $(T, 64)$ — one 64-d vector per token, per role.

The dimension shrinks from 768 to 64 because (as we'll see) we'll run 12 of these in parallel — each head has $64 \times 12 = 768$ total capacity.

Computing attention scores

Now: token $i$ asks its query $Q_i$ ("What am I looking for?"). Token $j$ answers with its key $K_j$ ("Here's what I have"). We measure compatibility with a dot product:

$$\text{score}(i, j) = Q_i \cdot K_j$$

A high dot product means "token $j$'s key aligns with token $i$'s query." After scaling and softmax, these scores become weights that sum to 1:

The attention equation
$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V$$
$QK^\top$ Matrix of all dot products. Shape $(T, T)$. Entry $(i,j)$ = how much token $i$ attends to token $j$.
$\sqrt{d_k}$ Scaling by $\sqrt{64} = 8$. Prevents dot products from growing so large they push softmax into saturation (near-zero gradients).
softmax Applied row-wise: each row sums to 1. Row $i$ is now a probability distribution over all positions $j$.
$(\cdots) V$ Weighted sum of value vectors. Token $i$'s output = $\sum_j w_{ij} V_j$ — a blend of all tokens' values, weighted by how much $i$ attends to each.

Concretely: if token $i$ is "bank" and the attention weights for "river" $(j=3)$ and "fish" $(j=7)$ are high, then the output for "bank" is dominated by those tokens' value vectors. The representation of "bank" is updated to reflect its riverbank context. The model has read the surrounding words.

Causal masking: no peeking ahead

During training, we predict the next token at every position simultaneously (that's how we can train on so much data efficiently). But token at position $i$ must not see tokens at positions $> i$ — that would be cheating, looking at the answer. So before the softmax, we set future positions to $-\infty$:

$$\text{score}(i,j) = -\infty \quad \text{if } j > i$$

After softmax, $e^{-\infty} = 0$, so those positions get zero weight. This is called the causal mask, and it's the reason transformers are called "causal" or "decoder-only." It ensures predictions are always made from past tokens only.

Multi-head attention

One attention head can learn one type of relationship — maybe syntactic subject-verb agreement. But "Paris" relates to "capital" semantically, to "is" grammatically, and to "France" geographically — all at once.

GPT-2 runs 12 attention heads in parallel, each with its own $W_Q^h, W_K^h, W_V^h$ and $d_k = 64$. Each head attends to the sequence differently. Their outputs (each $(T, 64)$) are concatenated to get $(T, 768)$ and projected back to the model dimension:

$$\text{MultiHead}(X) = \text{concat}(\text{head}_1, \ldots, \text{head}_{12}) \, W_O$$

where $W_O$ is a $(768, 768)$ projection matrix. The full attention sub-layer is 12 different "lenses" looking at the sequence, combined into one output.

5. After attention: the MLP

Attention has done something remarkable: it updated each token's representation by reading from every other relevant token. The representation of "bank" now reflects "river" and "fish." But attention is still a linear operation over the token vectors — it blends them, but the blending is a linear function of the inputs.

What if the right "understanding" of a word is a complex, non-linear function of its context? That's what the MLP (multi-layer perceptron) handles. After attention, every token position independently runs through a two-layer neural network:

The MLP sub-layer (per token, independently)
$$\text{MLP}(x) = W_2 \cdot \text{GELU}(W_1 x + b_1) + b_2$$
$W_1$ Expands $x$ from $768 \to 3072$ (four times the model size). Learned.
$\text{GELU}$ A smooth nonlinearity (like ReLU but differentiable everywhere). Creates non-linear representations.
$W_2$ Projects back from $3072 \to 768$. Learned.

Think of it as: attention decides what to gather from context; MLP decides what to make of it — encoding higher-level semantic facts. The expansion to 3072 gives the model room to represent complex intermediate features before projecting back.

Where facts live

Recent research suggests the MLP weights store factual knowledge — things like "Paris → capital → France" — while attention handles structure and relationships. When you hear "GPT hallucinates," one mechanism is the MLP weights encoding incorrect associations from training data.

6. The information highway: residual connections

Here's a problem that arises when you stack 12 of these blocks (attention + MLP + attention + MLP + …): the signal from the original embedding gets transformed so many times that gradients struggle to flow all the way back to the first layer. The network becomes hard to train. Early layers stop learning.

The fix, discovered for ResNet image models in 2015 and adopted everywhere since, is the residual connection (also called a skip connection). Instead of replacing $x$ with the transformation, you add the transformation to $x$:

One transformer block, precisely
$$x \;\leftarrow\; x + \text{Attention}(\text{LayerNorm}(x))$$ $$x \;\leftarrow\; x + \text{MLP}(\text{LayerNorm}(x))$$
$x + \cdots$ The "residual": add the transformation to the input. If the transformation outputs zero, $x$ passes through unchanged — a safe default.
LayerNorm Normalizes $x$ to zero mean and unit variance before attention/MLP. Keeps activations in a stable range so gradients don't explode or vanish.

Why does this help training? The gradient of the residual path is: $\frac{\partial (x + f(x))}{\partial x} = 1 + \frac{\partial f}{\partial x}$. The $1$ term is a direct gradient highway that bypasses the transformation entirely. Gradients flow cleanly from the last block back to the first, even through 12 or 96 layers.

7. The full picture: gpt() from input to logits

Now we can trace the complete forward pass of gpt() — microgpt.py lines 73–143:

1
Token + positional embeddings
Token IDs → wte lookup → token vectors, shape $(T, 768)$.
Positions $0, 1, \ldots, T-1$ → wpe lookup → position vectors, shape $(T, 768)$.
Add element-wise: $X = \text{wte}[\text{ids}] + \text{wpe}[\text{positions}]$.
2
Stack of 12 transformer blocks
Each block: x = x + Attention(LN(x)), then x = x + MLP(LN(x)).
After 12 blocks: $X$ still has shape $(T, 768)$, but now every token's vector is richly informed by the full context.
3
Final LayerNorm
One last normalization pass over the output of the last block.
4
Linear head → logits
Multiply by lm_head, shape $(768, 50257)$ — one number per vocab item.
Output: shape $(T, 50257)$. Row $i$ = logits for position $i$. Softmax → probability distribution over 50,257 tokens. 
def gpt(token_ids, positions):
    # Step 1: embeddings (microgpt lines ~73–85)
    tok_emb = wte[token_ids]          # (T, 768)
    pos_emb = wpe[positions]          # (T, 768)
    x = tok_emb + pos_emb             # (T, 768)

    # Step 2: transformer blocks (lines ~86–125)
    for block in blocks:              # 12 blocks for GPT-2
        x = x + attn(layer_norm(x))   # attention + residual
        x = x + mlp(layer_norm(x))    # MLP + residual

    # Step 3: final norm + head (lines ~126–143)
    x = layer_norm_final(x)           # (T, 768)
    logits = lm_head(x)               # (T, 50257)
    return logits

Notice: the shape is $(T, 768)$ from step 1 all the way through step 3. The transformer blocks update the values inside the vectors — they don't change the shape. Only the final linear head changes it to $(T, 50257)$.

The analogy

Embeddings give each token a sticky note with initial information. Each transformer block lets every token read and update every other token's sticky note. After 12 rounds of reading and updating, each sticky note has been enriched by the full context. The linear head then turns the final sticky note at position $T-1$ into a probability distribution over what comes next.

Q1 — The embedding matrix wte has shape $(50257, 768)$. What does the number 50257 represent?

Correct. One row per vocabulary token. Looking up token ID 2318 retrieves row 2318 — a 768-dimensional vector. The 768 is the embedding dimension, not the vocab size.

Not quite. The rows of the embedding matrix are indexed by token ID — so the number of rows equals the vocabulary size. 50,257 = size of GPT-2's vocabulary.

Q2 — A sentence has 6 tokens. After the embedding step but before the attention blocks, what is the shape of the data matrix $X$?

Correct. $(T, d_{\text{model}}) = (6, 768)$. This shape persists all the way through all 12 transformer blocks — only the final linear head changes it to $(6, 50257)$.

Not quite. The embedding step produces one 768-dimensional vector per token. For $T = 6$ tokens, that's a $(6, 768)$ matrix. The $(T, 50257)$ shape only appears at the very end, after the linear head.

Q3 (hard) — Attention weights for token $i$ over all positions $j$ sum to 1.0. What would it mean if all 6 weights were exactly $\frac{1}{6}$?

Correct. Uniform attention means the query $Q_i$ doesn't have a strong dot-product match with any particular key. The output is just the average of all value vectors — no contextual sharpening. High-entropy attention = the model didn't find a strongly relevant neighbor. Sharp attention (one weight near 1.0) = strong, specific dependency found.

Think about what uniform weights mean computationally: output = $\frac{1}{6}(V_1 + \ldots + V_6)$ — a plain average of all value vectors. The model learned nothing specific about which tokens matter. This is a diffuse, low-information pattern — the opposite of a sharp, focused dependency.

SR1 — Describe the attention mechanism in one sentence: what goes in, what comes out, and what does it compute?

Each token projects its embedding into Query, Key, and Value vectors; dot products between every Query and Key produce attention weights (softmaxed to sum to 1); the output for each token is a weighted sum of all Value vectors, so each token's representation is updated by the context it attends to most strongly.

SR2 — What specific problem does the residual connection solve? State the mechanism that makes it work.

Gradient vanishing in deep networks — gradients shrink to near-zero as they propagate back through many layers, so early layers stop learning. The residual mechanism adds 1 to the gradient: $\frac{\partial}{\partial x}(x + f(x)) = 1 + \frac{\partial f}{\partial x}$. The +1 is a direct highway for gradients, independent of the layer's transformation.
