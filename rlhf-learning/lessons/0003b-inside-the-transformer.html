Q1 (Lesson 3) — What does the training loop do to the model's parameters at each step? Name the five-stage loop.

Forward → compute the model's prediction → Loss → measure how wrong it was (cross-entropy: $-\log P(y_\text{correct})$) → Zero grad → clear stale gradients → Backward → backprop: compute $\frac{\partial L}{\partial \theta_i}$ for every parameter → Update → nudge every parameter downhill: $\theta \leftarrow \theta - \eta \frac{\partial L}{\partial \theta}$.

Q2 (Lesson 2 + 3) — What does gpt() take as input, and what does it return? What type is each?

Input: a list of token IDs — integers (e.g. [15496, 11, 314]). Output: logits — one floating-point number per vocabulary item, at each position. Softmax turns logits into a probability distribution. The model doesn't know what "Paris" means; it only sees the integer ID for it.

1. The problem with integers

Every lesson so far, we've fed gpt() a list of token IDs — integers like 15496 for "Hello." We never asked: what happens inside?

Let's start with a fact that should bother you.

The integer problem

Token IDs are arbitrary integers. "cat" = 2318. "dog" = 3290. "kitten" = 19090.

But "cat" and "kitten" mean almost the same thing! Yet their IDs are 16,772 apart. And "cat" (2318) is closer in number to "car" (2315) — a completely unrelated word. The numbers are random labels. You can't do meaningful math on them.

But the transformer has to do math. So what does it do?

Think of it this way

Imagine you're organizing a library, and every book has a random barcode number. You can't tell from the barcodes that two books are both about cooking. What you need is a description card for each book — a list of properties like "topic: food, difficulty: easy, language: English." Now similar books have similar description cards, even if their barcodes are nothing alike.

That's exactly what the transformer does. It replaces each meaningless ID with a rich description vector — 768 numbers that do capture meaning. Let's see how.

2. Embeddings: giving each token a meaningful address

The model keeps a giant lookup table called the embedding matrix (wte in microgpt.py — "weight token embedding"). Think of it as a dictionary: you give it a token ID, it gives you back a list of 768 numbers.

Let's make it tiny to see how it works

Forget 768 dimensions — imagine our embedding only has 3 numbers per word, and our vocabulary is only 5 words. The entire embedding table might look like this:

Token IDTokenEmbedding vector (3 numbers)
0"cat"[ 0.9, 0.1, 0.8 ]
1"dog"[ 0.8, 0.2, 0.7 ]
2"car"[ 0.1, 0.9, 0.2 ]
3"kitten"[ 0.85, 0.15, 0.75 ]
4"truck"[ 0.15, 0.85, 0.25 ]

Look at the numbers. "cat" is [0.9, 0.1, 0.8] and "kitten" is [0.85, 0.15, 0.75] — almost the same! Meanwhile "car" is [0.1, 0.9, 0.2] — totally different from "cat," even though their IDs (0 and 2) are close. The numbers encode meaning, not alphabetical order.

Maybe the first number captures "animal-ness," the second captures "vehicle-ness," and the third captures something else. We didn't program this — the model learned these numbers through training. Gradient descent naturally pushed similar words toward similar vectors, because words that appear in similar contexts should have similar representations.

Worked example: looking up an embedding
Input:
The sentence "cat dog" → token IDs [0, 1]
Lookup:
ID 0 → row 0 of the table → [0.9, 0.1, 0.8]
ID 1 → row 1 of the table → [0.8, 0.2, 0.7]
Result:
Our input matrix $X$ is now: $\begin{bmatrix} 0.9 & 0.1 & 0.8 \\ 0.8 & 0.2 & 0.7 \end{bmatrix}$ — shape $(2, 3)$.
In real GPT-2, this would be shape $(T, 768)$. Same idea, bigger numbers.
The surprising consequence

Because similar words end up with similar vectors, you can do meaningful arithmetic on them. The famous example: vec("king") − vec("man") + vec("woman") ≈ vec("queen"). Nobody programmed this — it emerged from predicting the next word on billions of sentences.

So: integer ID → table lookup → meaningful vector. Now the model has something it can do real math on. But there's a second problem.

3. Position matters: the second embedding

Here's a puzzle. Consider these two sentences:

"The dog bit the man" "The man bit the dog"

Same exact words. Completely different meaning. One person is having a very bad day. The only difference is which word goes where.

But our embedding lookup doesn't know about position! It always gives the same vector for "dog" whether it's the first word or the last. The model would be completely blind to word order.

The fix is beautifully simple

Add a second lookup table — one that encodes position. Position 0 gets its own vector. Position 1 gets a different one. And so on. Then you just add the two vectors together: "what the word is" + "where it sits."

Worked example: adding position information

Continuing our tiny 3-dimensional world. Say the position table looks like:

PositionPosition vector
0 (first word)[0.1, 0.0, 0.1]
1 (second word)[0.0, 0.1, 0.0]
For "cat dog" (IDs [0, 1]):
"cat" at position 0: [0.9, 0.1, 0.8] + [0.1, 0.0, 0.1] = [1.0, 0.1, 0.9]
"dog" at position 1: [0.8, 0.2, 0.7] + [0.0, 0.1, 0.0] = [0.8, 0.3, 0.7]
For "dog cat" (IDs [1, 0]):
"dog" at position 0: [0.8, 0.2, 0.7] + [0.1, 0.0, 0.1] = [0.9, 0.2, 0.8]
"cat" at position 1: [0.9, 0.1, 0.8] + [0.0, 0.1, 0.0] = [0.9, 0.2, 0.8]

Now "dog-at-position-0" and "dog-at-position-1" have different vectors. The model can tell them apart. Problem solved.

Input to the transformer stack
$$x_i = \underbrace{\text{wte}[\text{token\_id}_i]}_{\text{what the word is}} + \underbrace{\text{wpe}[i]}_{\text{where it sits}}$$

For $T$ tokens, we get a matrix $X$ of shape $(T, 768)$. In our toy example, $(2, 3)$. This is what enters the transformer blocks.

Both tables are learned — the model figures out how to encode meaning and position through training. We just provide the lookup; gradient descent fills in the numbers.

Now we have a rich vector for each token that captures both what it is and where it sits. The transformer's job is to let each token look at the others and update itself. Which brings us to the core idea.

4. Self-attention: the core idea

Here's the problem we need to solve. The word "bank" has one embedding — one fixed vector. But "bank" means completely different things here:

"She walked to the river bank to fish." "She called her investment bank about the loan."

A fixed vector for "bank" can't capture both meanings. What we need is a way for "bank" to look around at the other words and update itself. When it sees "river" and "fish" nearby, it should shift toward its riverbank meaning. When it sees "investment" and "loan," it should shift toward its financial meaning.

That's attention. Let's build it from scratch with a tiny example.

The party analogy

Think of a cocktail party

You walk into a party. You're interested in talking about cooking (that's your Query — what you're looking for).

Every other person at the party is wearing a name tag that says what they know about (that's their Key — what they advertise).

You scan the room. The person whose name tag says "chef" — high match with your interest! The person whose tag says "accountant" — low match. You walk over to the chef.

When you talk to the chef, they share their actual knowledge with you (that's their Value — the content they contribute).

Now imagine you don't just talk to one person — you talk to everyone, but you pay more attention to the ones whose tags match your interest. You come away with a blend of information, weighted by relevance.

That's attention. Every token is simultaneously a partygoer with a question (Query), a name tag (Key), and knowledge to share (Value).

Step 1: Create Q, K, V from the embeddings

Each token's embedding gets transformed into three separate vectors via matrix multiplication. Let's see this with actual numbers.

Worked example: computing Q, K, V for 3 tokens

Let's say we have the sentence "The cat sat" and (to keep it tiny) our embeddings are just 2 numbers each:

TokenEmbedding
"The"[1, 0]
"cat"[0, 1]
"sat"[1, 1]

The model has learned weight matrices that transform each embedding into a Query, Key, and Value. Suppose the learned weights are:

$W_Q$ (query weights):
$\begin{bmatrix} 1 & 0 \\ 0 & 1 \end{bmatrix}$ — in this toy case, it happens to be the identity (in real life it's a learned 768×64 matrix)
$W_K$ (key weights):
$\begin{bmatrix} 0 & 1 \\ 1 & 0 \end{bmatrix}$ — this swaps the two values
$W_V$ (value weights):
$\begin{bmatrix} 1 & 1 \\ 0 & 1 \end{bmatrix}$

Now we multiply each embedding by each weight matrix:

TokenQuery (embedding × $W_Q$)Key (embedding × $W_K$)Value (embedding × $W_V$)
"The" [1,0][1, 0][0, 1][1, 1]
"cat" [0,1][0, 1][1, 0][0, 1]
"sat" [1,1][1, 1][1, 1][1, 2]

Now each token has three roles: what it's looking for (Q), what it advertises (K), and what it'll share (V).

Step 2: Compute attention scores (who should talk to whom?)

Here's the key insight: to find out how much token A should attend to token B, we take the dot product of A's Query with B's Key. A high dot product = "B has what A is looking for."

What does the dot product actually measure?

The dot product of two vectors measures how much they point in the same direction. If I'm looking for [1, 0] (pointing east) and your name tag says [1, 0] (also east) — dot product is $1 \times 1 + 0 \times 0 = 1$. High match!

If your tag says [0, 1] (pointing north) — dot product is $1 \times 0 + 0 \times 1 = 0$. No match at all.

It's literally asking: "are these two vectors aligned?"

Worked example: computing all attention scores

Let's compute how much "cat" (Query = [0, 1]) attends to each token:

"cat" Q · "The" K:
[0, 1] · [0, 1] = 0×0 + 1×1 = 1 — moderate match
"cat" Q · "cat" K:
[0, 1] · [1, 0] = 0×1 + 1×0 = 0 — no match (cat doesn't find itself interesting here!)
"cat" Q · "sat" K:
[0, 1] · [1, 1] = 0×1 + 1×1 = 1 — moderate match

The full score matrix for all tokens (every Query dotted with every Key):

"The" K"cat" K"sat" K
"The" Q011
"cat" Q101
"sat" Q112

Read row by row: "The" scores [0, 1, 1] — it finds "cat" and "sat" relevant. "sat" scores [1, 1, 2] — it finds itself most interesting (score 2).

Step 3: Scale and softmax (turn scores into percentages)

Raw dot products can be big numbers (especially with 64-dimensional vectors). Big numbers make softmax "peaky" — it would put nearly 100% weight on one token and ignore the rest. So we divide by $\sqrt{d_k}$ to keep things gentle.

Then softmax turns each row of scores into probabilities that sum to 1.

Worked example: softmax on "cat"'s scores
Raw scores for "cat":
[1, 0, 1]
Scale by $\sqrt{d_k}$ (in our toy case $\sqrt{2} \approx 1.41$):
[0.71, 0.00, 0.71]
Softmax — compute $e^x$ for each, then normalize:
$e^{0.71} = 2.03, \quad e^{0.00} = 1.00, \quad e^{0.71} = 2.03$
Sum = $2.03 + 1.00 + 2.03 = 5.06$
Weights = [2.03/5.06, 1.00/5.06, 2.03/5.06] = [0.40, 0.20, 0.40]

So "cat" puts 40% attention on "The," 20% on itself, and 40% on "sat." These weights sum to 1.0 — it's a probability distribution.

Step 4: Weighted sum of Values (the actual output)

Now we use those attention weights to blend the Value vectors. The output for "cat" is a weighted average of everyone's Values:

Worked example: computing "cat"'s new representation
Attention weights for "cat":
"The" = 0.40, "cat" = 0.20, "sat" = 0.40
Value vectors:
"The" V = [1, 1], "cat" V = [0, 1], "sat" V = [1, 2]
Weighted sum:
$0.40 \times [1, 1] + 0.20 \times [0, 1] + 0.40 \times [1, 2]$
$= [0.4, 0.4] + [0.0, 0.2] + [0.4, 0.8]$
$= \mathbf{[0.8, 1.4]}$

This is "cat"'s new representation — no longer just its original embedding, but a blend that's been informed by its context. "cat" has read the other words and updated itself.

The full equation (now you can read it)

Everything we just did by hand is captured in one formula:

The attention equation
$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V$$
$QK^\top$ The score matrix — all dot products between Queries and Keys. We computed this by hand above.
$\div \sqrt{d_k}$ Scaling to keep scores from getting too large. We divided by $\sqrt{2}$ above; in GPT-2 it's $\sqrt{64} = 8$.
softmax Turns each row of scores into weights that sum to 1 — the "how much to attend" percentages.
$\times\, V$ Weighted sum of Value vectors — the actual output. Each token gets a new representation that blends in context.
The one-sentence summary

Each token asks "who is relevant to me?" (Q·K scores), turns those scores into percentages (softmax), and then takes a weighted average of everyone's content (V). The output is a new vector that's been enriched by context.

Causal masking: no peeking ahead

There's a rule: during training, a token can only look at words that came before it. Otherwise it would be cheating — like reading the answer before writing your response.

Worked example: causal masking

In "The cat sat", before softmax, we set scores for future positions to $-\infty$:

"The""cat""sat"
"The"0$-\infty$$-\infty$
"cat"10$-\infty$
"sat"112

"The" can only see itself. "cat" can see "The" and itself. "sat" can see everyone.

After softmax, $e^{-\infty} = 0$, so masked positions get exactly zero attention. "The" puts 100% weight on itself — it has no context at all. That's the rule: you can only read backward.

Multi-head attention: 12 different perspectives

One attention head learns one type of relationship. But words relate to each other in many ways at once. "The cat sat" — "cat" relates to "The" grammatically (it's the noun after the article) and to "sat" semantically (cats sit). A single head might only catch one of these.

The key idea: split, look separately, combine

Imagine you're reading a sentence and you have two friends helping you. You say: "Friend 1, focus on grammar — who's the subject, who's the verb. Friend 2, focus on meaning — which words are related in concept." Each friend looks at the same sentence but notices different things. Then you combine their notes into one summary.

That's multi-head attention. GPT-2 has 12 "friends" (heads), each with their own Q, K, V weights, each looking for a different pattern. But here's the trick that makes the math work: instead of giving each head all 768 dimensions, you split the 768 into 12 chunks of 64. Each head works with 64 dimensions — smaller, cheaper, and forced to specialize.

Let's see exactly how this works with numbers. We'll use 2 heads instead of 12 (same idea, easier to follow).

Worked example: 2 attention heads on "The cat sat"

Our tokens have 4-dimensional embeddings (instead of 768). With 2 heads, each head gets $4 \div 2 = 2$ dimensions to work with.

TokenFull embedding (4-d)
"The"[1, 0, 0, 1]
"cat"[0, 1, 1, 0]
"sat"[1, 1, 0, 1]
Step A: Each head gets its own Q, K, V weight matrices
Head 1 has $W_Q^1, W_K^1, W_V^1$ — each is $(4, 2)$, projecting from 4-d down to 2-d.
Head 2 has $W_Q^2, W_K^2, W_V^2$ — also $(4, 2)$, but different learned values.

Think of it as: each head gets its own pair of glasses. Same sentence, different things light up.
Step B: Head 1 runs attention (focuses on grammar)
Head 1's weights produce these Q, K, V for each token:
TokenQ1K1V1
"The"[1, 0][0, 1][1, 0]
"cat"[0, 1][1, 1][0, 1]
"sat"[1, 1][0, 1][1, 1]
Scores for "cat" (Q=[0,1]): · "The" K=[0,1] → 1, · "cat" K=[1,1] → 1, · "sat" K=[0,1] → 1
After softmax: [0.33, 0.33, 0.33] — uniform attention. Head 1 doesn't find strong preferences for "cat".
Output for "cat": $0.33 \times [1,0] + 0.33 \times [0,1] + 0.33 \times [1,1] = \mathbf{[0.67, 0.67]}$
Step C: Head 2 runs attention (focuses on meaning)
Head 2's different weights produce different Q, K, V:
TokenQ2K2V2
"The"[0, 1][1, 0][0, 0]
"cat"[1, 0][0, 1][1, 1]
"sat"[1, 1][1, 1][0, 1]
Scores for "cat" (Q=[1,0]): · "The" K=[1,0] → 1, · "cat" K=[0,1] → 0, · "sat" K=[1,1] → 1
After softmax: [0.42, 0.16, 0.42] — head 2 focuses on "The" and "sat," ignores self!
Output for "cat": $0.42 \times [0,0] + 0.16 \times [1,1] + 0.42 \times [0,1] = \mathbf{[0.16, 0.58]}$
Step D: Concatenate the heads
Head 1 gave "cat" → [0.67, 0.67] (2 numbers)
Head 2 gave "cat" → [0.16, 0.58] (2 numbers)

Glue them together: [0.67, 0.67, 0.16, 0.58] — back to 4 numbers!

This is why $64 \times 12 = 768$ in real GPT-2. Each head outputs 64 numbers, you concatenate all 12, and you get 768 back. The total capacity is the same — you've just split the work into 12 specialists.
Step E: Project with $W_O$
One final matrix multiplication: $W_O$ is $(4, 4)$. It mixes the heads' outputs together:

[0.67, 0.67, 0.16, 0.58] × $W_O$ → [0.8, 1.1, 0.4, 0.9]

This is the final attention output for "cat" — a single 4-d vector that combines what both heads learned. Head 1 contributed its grammatical reading, head 2 contributed its semantic reading, and $W_O$ blended them.
Why not just use one big head?

You could. One head with 768 dimensions would have the same number of parameters. But it would try to do everything at once — grammar, meaning, position — with a single set of attention scores. It's like asking one person to simultaneously evaluate a restaurant's food, service, and ambiance with a single score. Much better to have three reviewers, each scoring one aspect, then combine.

Multi-head = multiple specialists > one generalist.

$$\text{MultiHead}(X) = \text{concat}(\text{head}_1, \ldots, \text{head}_{12}) \, W_O$$

Each head runs the full Q→K→V→scores→softmax→weighted-sum pipeline independently. Their results are concatenated and projected by $W_O$. The whole thing has shape $(T, 768)$ in, $(T, 768)$ out.

5. After attention: the MLP

Attention did something beautiful: "bank" read the words "river" and "fish" and updated itself to lean toward "riverbank." But all attention does is blend existing vectors — it's a weighted average, which is a linear operation. Averages can't capture complex, nonlinear patterns.

Why do we need more than blending?

Imagine you know that "river" is nearby and "bank" is nearby. To conclude "this is about geography, not finance," you need to do something more than just average the two vectors — you need to reason about the combination. That reasoning requires nonlinearity. A weighted average of "river" and "bank" is still just a point between them. You need a function that can say "river + bank → aha, this is a riverbank, activate a whole new cluster of meaning."

That's the MLP's job. It's a tiny two-layer neural network that processes each token independently after attention has mixed in the context.

Worked example: MLP on a single token

Let's say after attention, "cat"'s vector is [0.8, 1.4] (from our example). The MLP does three things:

1. Expand: multiply by $W_1$ to make the vector bigger
In our toy: $W_1$ takes 2 numbers → 4 numbers. Say the result is [0.8, 1.4, -0.2, 2.1].
In real GPT-2: 768 → 3072 (four times bigger). More room to think.
2. Apply nonlinearity (GELU): squash negative values, keep positive ones
[0.8, 1.4, -0.2, 2.1] → roughly [0.7, 1.4, -0.02, 2.1]
The -0.2 got nearly zeroed out. This is where nonlinear "reasoning" happens.
3. Compress: multiply by $W_2$ to shrink back to original size
4 numbers → 2 numbers. Say the result is [1.1, 0.9].
In real GPT-2: 3072 → 768.

The token went in as [0.8, 1.4] and came out as [1.1, 0.9] — transformed by a nonlinear function that captured something about the combination of context gathered by attention.

The MLP sub-layer (per token, independently)
$$\text{MLP}(x) = W_2 \cdot \text{GELU}(W_1 x + b_1) + b_2$$
$W_1$ Expands from $768 \to 3072$. Like giving the model a bigger scratchpad.
$\text{GELU}$ The nonlinearity — squashes some values, keeps others. This is where "understanding" happens.
$W_2$ Compresses back from $3072 \to 768$. Distills the result.
The division of labor

Attention = gather information from context ("look around the room").
MLP = process what you gathered ("think about what you heard").
One gathers, the other reasons. Together they form one transformer block.

Where facts live

Recent research suggests the MLP weights store factual knowledge — things like "Paris → capital → France" — while attention handles structure and relationships. When you hear "GPT hallucinates," one mechanism is the MLP weights encoding incorrect associations from training data.

5b. LayerNorm: keeping the numbers sane

You've seen "LayerNorm" pop up in the transformer block diagram, and maybe wondered: what is that, and why do we need it? Let's answer both questions.

The problem: numbers drift out of control

Why numbers go haywire

Each transformer block does a bunch of matrix multiplications and additions. After a few blocks, some numbers in a token's vector might grow to 500 while others shrink to 0.001. This is a problem because:

1. Softmax breaks. If the inputs to softmax are [500, 501, 499], then $e^{500}$ is astronomically large. The softmax becomes nearly [0, 1, 0] — it puts all weight on one token and ignores everything else. The model can't learn nuanced attention patterns.

2. Gradients break. When numbers are huge, gradients become either huge (exploding) or tiny (vanishing). Training becomes unstable — the model oscillates wildly or stops learning.

The fix: before each attention and MLP step, normalize the numbers to a nice range. That's LayerNorm.

How LayerNorm works: 3 simple steps

LayerNorm operates on each token's vector independently. It asks: "What's the average of these numbers? How spread out are they?" Then it centers them at 0 and scales them to have a standard spread of 1.

Worked example: LayerNorm on one token vector

Suppose after a few transformer blocks, "cat"'s vector has drifted to:

[10, 2, 6, 14]

These numbers are all over the place. Let's normalize them.

Step 1: Compute the mean (average)
$\mu = \frac{10 + 2 + 6 + 14}{4} = \frac{32}{4} = 8$
Step 2: Subtract the mean (center at zero)
$[10 - 8,\; 2 - 8,\; 6 - 8,\; 14 - 8] = [2, -6, -2, 6]$
Now the values are balanced around 0.
Step 3: Divide by the standard deviation (scale to unit spread)
Variance = $\frac{2^2 + (-6)^2 + (-2)^2 + 6^2}{4} = \frac{4 + 36 + 4 + 36}{4} = \frac{80}{4} = 20$

Standard deviation = $\sqrt{20} \approx 4.47$

$\left[\frac{2}{4.47},\; \frac{-6}{4.47},\; \frac{-2}{4.47},\; \frac{6}{4.47}\right] = \mathbf{[0.45, -1.34, -0.45, 1.34]}$

The vector went from [10, 2, 6, 14] (mean 8, spread all over) to [0.45, -1.34, -0.45, 1.34] (mean 0, standard deviation 1). The relative pattern is preserved — the 4th number is still the biggest, the 2nd is still the smallest — but the scale is now tidy and predictable.

LayerNorm formula
$$\text{LayerNorm}(x) = \gamma \odot \frac{x - \mu}{\sigma + \epsilon} + \beta$$
$\mu, \sigma$ Mean and standard deviation of the vector's elements. Computed fresh for each token, every time.
$\epsilon$ A tiny number (like 0.00001) added to $\sigma$ so we never divide by zero.
$\gamma, \beta$ Learned scale and shift parameters. After normalizing, the model can learn to re-scale if it wants to. These are trained just like any other weight.
An analogy for LayerNorm

Imagine each token's vector is a set of test scores from different subjects: Math = 95, Art = 30, Science = 70. The raw numbers are hard to compare — is 30 in Art bad? Maybe Art is graded on a tighter scale.

LayerNorm is like converting everything to "how many standard deviations from average." Now Math = +1.2, Art = -1.1, Science = +0.1. The relative ordering is preserved, but the numbers are on a common scale. The neural network can work with these clean numbers much more easily.

Where LayerNorm appears in the block

In GPT-2, LayerNorm happens before each sub-layer (attention and MLP). The sequence is:

The full pipeline for one transformer block
1. LayerNorm the input
Normalize "cat"'s vector: [10, 2, 6, 14] → [0.45, -1.34, -0.45, 1.34]
2. Run attention on the normalized vector
Attention produces an update: [0.3, 0.8, -0.1, 0.5]
3. Add residual (add update to the original input, not the normalized one)
[10, 2, 6, 14] + [0.3, 0.8, -0.1, 0.5] = [10.3, 2.8, 5.9, 14.5]
4. LayerNorm again
Normalize [10.3, 2.8, 5.9, 14.5] → nice tidy numbers
5. Run MLP on the normalized vector
MLP produces an update: [0.1, -0.3, 0.2, 0.4]
6. Add residual again
[10.3, 2.8, 5.9, 14.5] + [0.1, -0.3, 0.2, 0.4] = [10.4, 2.5, 6.1, 14.9]

This is the output of one transformer block. It goes into the next block as input, where the whole process repeats.

Notice: LayerNorm normalizes before each operation, so attention and MLP always receive well-behaved inputs. But the residual connection adds to the un-normalized input, preserving the full information.

6. The information highway: residual connections

Now that we've seen all the individual pieces (attention, MLP, LayerNorm), we're going to stack 12 of them. But this creates a serious problem. Let's see it with numbers.

The problem: signals die in deep networks

Worked example: why 12 blocks without residuals would fail

Imagine each block's transformation slightly shrinks the gradient by a factor of 0.8 (which is realistic for a layer that multiplies by a weight matrix).

Gradient flowing backward through 12 blocks:
Start with gradient = 1.0 at the output
After block 12: $1.0 \times 0.8 = 0.80$
After block 11: $0.80 \times 0.8 = 0.64$
After block 10: $0.64 \times 0.8 = 0.51$
...
After block 1: $0.8^{12} = \mathbf{0.069}$

The gradient that reaches block 1 is only 6.9% of the original signal. Block 1 barely learns at all. Block 12 gets the full gradient and learns quickly, but block 1 is nearly frozen.

Now imagine a deeper model (96 blocks, like GPT-3):
$0.8^{96} = 0.000000021$ — the gradient is effectively zero. The first layers are completely dead.

This is the vanishing gradient problem. Before residual connections were invented (2015, by the ResNet team), it was nearly impossible to train networks deeper than ~20 layers. The fix is so simple it's almost surprising it works.

The fix: add, don't replace

Instead of passing $x$ through a transformation and replacing it with the result, you add the result to the original:

Without residual: $x \to f(x)$. The original $x$ is gone.

With residual: $x \to x + f(x)$. The original $x$ always survives.

Why does this fix gradients? Because the gradient of $x + f(x)$ with respect to $x$ is $1 + \frac{\partial f}{\partial x}$. Even if $\frac{\partial f}{\partial x}$ is tiny, that +1 is always there — a direct highway that carries the gradient from the last block to the first, untouched.

Worked example: residual connection saves the signal
Before the attention block:
"cat"'s vector = [0.8, 0.3]
Attention produces:
[0.0, 1.1] (the update from reading other tokens)
Without residual connection:
New vector = [0.0, 1.1] — the first component (0.8) is gone forever. That information can never be recovered by later blocks.
With residual connection:
New vector = [0.8, 0.3] + [0.0, 1.1] = [0.8, 1.4] — original information preserved, new context layered on top.
Worked example: gradient highway through 12 blocks
With residuals, the gradient at block 1:
The gradient has a path that goes directly from block 12 to block 1 through the residual connections — adding 1 at every step. The gradient arriving at block 1 includes the term $1^{12} = 1$. The gradient is fully preserved.

Even the additional terms ($\frac{\partial f}{\partial x}$ from each block) get added on top. So the gradient at block 1 is actually richer than without residuals — it's $1 + (\text{contributions from all blocks})$.

This is why residual connections are essential, not optional. Without them, deep transformers simply cannot be trained. Remove them and the model doesn't just "train slower" — the early layers stop learning entirely.

One transformer block, precisely
$$x \;\leftarrow\; x + \text{Attention}(\text{LayerNorm}(x))$$ $$x \;\leftarrow\; x + \text{MLP}(\text{LayerNorm}(x))$$
$x + \cdots$ The "residual": add the update to the input. If the transformation outputs zero, $x$ passes through unchanged — a safe default.
LayerNorm Normalizes $x$ before each sub-layer (as we just learned in 5b). Keeps numbers stable so attention and MLP get clean inputs.

The gradient highway: $\frac{\partial (x + f(x))}{\partial x} = 1 + \frac{\partial f}{\partial x}$. The +1 carries gradients from the very last block to the very first, no matter how deep the network.

7. The full picture: gpt() from input to logits

Let's put it all together. We've seen each piece — now here's how they connect. We'll trace "The cat sat" all the way through:

1
Token + positional embeddings
"The cat sat" → token IDs [1, 2, 3] → look up each in the embedding table → 3 vectors.
Add position vectors: position 0, 1, 2 → each token now encodes what it is + where it sits.
Shape: $(3, 768)$. In our toy example: $(3, 2)$.
2
Stack of 12 transformer blocks
Each block does: attention (gather context from other tokens) + residual, then MLP (reason about gathered context) + residual.
After 12 blocks: still shape $(3, 768)$, but now "cat" knows about "The" and "sat." Every token has read the whole sequence.
3
Final LayerNorm
One last normalization to keep numbers stable. Shape unchanged.
4
Linear head → logits
Multiply by a $(768, 50257)$ matrix → one score per vocabulary word, at each position.
Output: $(3, 50257)$. The last row is "given 'The cat sat,' what comes next?" Softmax → probability distribution over 50,257 tokens. 
def gpt(token_ids, positions):
    # Step 1: embeddings (microgpt lines ~73–85)
    tok_emb = wte[token_ids]          # (T, 768)
    pos_emb = wpe[positions]          # (T, 768)
    x = tok_emb + pos_emb             # (T, 768)

    # Step 2: transformer blocks (lines ~86–125)
    for block in blocks:              # 12 blocks for GPT-2
        x = x + attn(layer_norm(x))   # attention + residual
        x = x + mlp(layer_norm(x))    # MLP + residual

    # Step 3: final norm + head (lines ~126–143)
    x = layer_norm_final(x)           # (T, 768)
    logits = lm_head(x)               # (T, 50257)
    return logits

Notice: the shape is $(T, 768)$ from step 1 all the way through step 3. The transformer blocks update the values inside the vectors — they don't change the shape. Only the final linear head changes it to $(T, 50257)$.

The big picture in plain English

Every word starts as a point in a 768-dimensional space (its embedding). It doesn't know what's around it.

Then it goes through 12 rounds of: (1) look around at other words and gather relevant info (attention), (2) think about what you gathered (MLP). Each round adds to what the word already knows (residual connection) rather than replacing it.

After 12 rounds, the word "cat" doesn't just mean "cat" anymore — it means "cat-that-sat-after-The." Its vector has been shaped by its entire context.

The final linear head takes the last token's enriched vector and asks: "based on everything this sequence has told you, what word comes next?" The answer: a probability for every word in the vocabulary.

Q1 — The embedding matrix wte has shape $(50257, 768)$. What does the number 50257 represent?

Correct. One row per vocabulary token. Looking up token ID 2318 retrieves row 2318 — a 768-dimensional vector. The 768 is the embedding dimension, not the vocab size.

Not quite. The rows of the embedding matrix are indexed by token ID — so the number of rows equals the vocabulary size. 50,257 = size of GPT-2's vocabulary.

Q2 — A sentence has 6 tokens. After the embedding step but before the attention blocks, what is the shape of the data matrix $X$?

Correct. $(T, d_{\text{model}}) = (6, 768)$. This shape persists all the way through all 12 transformer blocks — only the final linear head changes it to $(6, 50257)$.

Not quite. The embedding step produces one 768-dimensional vector per token. For $T = 6$ tokens, that's a $(6, 768)$ matrix. The $(T, 50257)$ shape only appears at the very end, after the linear head.

Q3 (hard) — Attention weights for token $i$ over all positions $j$ sum to 1.0. What would it mean if all 6 weights were exactly $\frac{1}{6}$?

Correct. Uniform attention means the query $Q_i$ doesn't have a strong dot-product match with any particular key. The output is just the average of all value vectors — no contextual sharpening. High-entropy attention = the model didn't find a strongly relevant neighbor. Sharp attention (one weight near 1.0) = strong, specific dependency found.

Think about what uniform weights mean computationally: output = $\frac{1}{6}(V_1 + \ldots + V_6)$ — a plain average of all value vectors. The model learned nothing specific about which tokens matter. This is a diffuse, low-information pattern — the opposite of a sharp, focused dependency.

SR1 — Describe the attention mechanism in one sentence: what goes in, what comes out, and what does it compute?

Each token projects its embedding into Query, Key, and Value vectors; dot products between every Query and Key produce attention weights (softmaxed to sum to 1); the output for each token is a weighted sum of all Value vectors, so each token's representation is updated by the context it attends to most strongly.

SR2 — What specific problem does the residual connection solve? State the mechanism that makes it work.

Gradient vanishing in deep networks — gradients shrink to near-zero as they propagate back through many layers, so early layers stop learning. The residual mechanism adds 1 to the gradient: $\frac{\partial}{\partial x}(x + f(x)) = 1 + \frac{\partial f}{\partial x}$. The +1 is a direct highway for gradients, independent of the layer's transformation.
